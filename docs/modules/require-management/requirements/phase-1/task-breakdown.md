## 2. 阶段二：流转门禁和权限

目标：实现主流程推进、回退、任务门禁和权限控制。

### 2.1 状态流转

- 定义需求主状态枚举。
- 定义状态顺序。
- 校验只允许上一状态或下一状态流转。
- 完成状态处理。
- 状态变化写入历史。

### 2.2 任务门禁

- 实现开发任务门禁。
- 实现测试任务门禁。
- 实现门禁文案。
- 需求列表返回门禁摘要。
- 需求详情返回完整门禁信息。

### 2.3 权限和数据范围

- 接入现有当前用户上下文。
- `DATA_ALL` 用户查看全部需求。
- 普通用户查看自己创建或参与的需求。
- 状态流转校验 `REQUIREMENT_TRANSITION`。
- 任务维护校验 `REQUIREMENT_TASK_EDIT`。
- 任务负责人允许更新自己的任务状态。

## 3. 阶段三：前端看板重构

目标：按照新原型实现需求看板、详情、任务池和状态流转。

### 3.1 API 封装

- 扩展 `frontend/src/api/requirements.ts` 类型。
- 新增看板查询方法。
- 新增详情查询方法。
- 新增更新需求方法。
- 新增状态流转方法。
- 新增任务增删改查方法。
- 新增历史查询方法。

### 3.2 需求看板

- 重构 `RequirementView.vue` 为看板布局。
- 实现状态筛选栏。
- 实现关键词搜索。
- 实现需求卡片。
- 实现优先级和状态标签。
- 实现门禁状态展示。

### 3.3 新建需求

- 实现新建需求弹窗。
- 补齐提出人、提出日期、提出迭代、上线迭代、优先级字段。
- 创建后刷新看板并选中新需求。

### 3.4 详情面板

- 展示需求基础信息。
- 展示主流程进度。
- 展示用例集和 Bug 摘要。
- 保留 XMind 上传和用例集入口。

### 3.5 任务面板

- 需求开发阶段展示开发任务池。
- 需求测试阶段展示测试任务池。
- 支持新增任务。
- 支持编辑任务状态。
- 支持删除任务。
- 展示完成数量。

### 3.6 流转面板

- 展示当前门禁结果。
- 支持上一状态。
- 支持下一状态。
- 门禁不通过时禁用下一状态按钮。
- 流转成功后刷新详情和看板。

## 4. 阶段四：联动和验证

目标：保证新需求管理不破坏现有用例、Bug 和权限能力。

### 4.1 用例联动

- 回归需求下上传 XMind。
- 回归需求下查看用例集。
- 回归用例集编辑和导出。

### 4.2 Bug 联动

- 回归需求下创建 Bug。
- 需求详情展示 Bug 数量。
- 产品验收进入完成时提示未关闭 Bug。

### 4.3 验证命令

后端：

```bash
mvn -f /Users/liangpengpeng/Project/testPlatform/pom.xml test
```

前端：

```bash

```

### 4.4 手工验证

- 使用 `admin / 123456` 登录。
- 创建项目。
- 创建需求。
- 查看需求进入“需求采集”。
- 正向流转到“需求开发”。
- 未创建开发任务时，下一状态不可用。
- 创建开发任务。
- 开发任务未完成时，不能进入测试。
- 全部开发任务完成后，进入测试。
- 创建测试任务。
- 测试任务未完成时，不能进入 UI 验收。
- 全部测试任务完成后，进入 UI 验收。
- 继续流转到产品验收和完成。
- 回退上一状态。
- 检查历史记录。
- 回归需求下 XMind 上传、用例编辑、Bug 创建。
