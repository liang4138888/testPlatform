需求交付流程图

根据实际交付流程整理：主流程由角色任务的完成情况驱动，开发、测试、验收等阶段支持多人并行和完成汇聚。

  testStart[进入测试]

  uiAccept[UI验收]

  start --> collect --> preReview --> detailReview --> assign --> devPool
  devPool --> devGate
  devGate -- 否 --> devPool
  devGate -- 是 --> testStart
  testStart --> tester
  tester --> testGate
  testGate -- 否 --> tester
  testGate -- 是 --> uiAccept --> productAccept --> done

流转规则

  1. 需求分配后，需求进入开发任务池，并按参与角色或人员生成独立开发任务。
  2. 同一角色可以存在多个任务，例如多个服务端开发并行处理不同模块。
  3. 不同角色任务并行推进，例如服务端、iOS、安卓、小程序、鸿蒙、Web 可以同步开发。
  4. 开发任务池内所有开发任务均为“开发完成”后，需求主流程才能进入测试。
  5. 测试、UI验收、产品验收也可以按角色或人员拆分任务，并通过“全部完成”规则控制主流程流转。
