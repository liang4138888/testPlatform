# Bug 管理模块需求文档

## 1. 背景

测试平台下一阶段需要补充 Bug 管理能力，用于记录需求验收、用例执行和测试过程中发现的问题。Bug 管理需要和现有项目、需求、用例集、文件能力打通，并支持图片上传、评论协作、操作历史、站内通知、登录认证、用户权限和指派人体系。

## 2. 目标

- 支持在需求下创建 Bug。
- Bug 编号由系统自动生成。
- Bug 必须关联需求，可选关联用例集。
- Bug 支持多张图片上传和预览。
- Bug 支持指派给系统用户。
- Bug 支持状态、严重级别、优先级维护。
- Bug 支持评论流。
- Bug 支持关键操作历史记录。
- Bug 关键协作动作支持站内通知。
- 预留外部通知扩展能力。
- 引入登录认证、用户、角色、权限体系。

## 3. 用户角色

### 管理员

- 管理用户、角色和权限。
- 查看、创建、编辑、关闭所有 Bug。
- 指派 Bug。
- 查看所有通知和历史。

### 测试人员

- 创建 Bug。
- 上传 Bug 图片。
- 评论 Bug。
- 编辑自己创建且未关闭的 Bug。
- 指派 Bug 给开发人员。

### 开发人员

- 查看指派给自己的 Bug。
- 修改 Bug 处理状态。
- 评论 Bug。
- 查看 Bug 图片、历史和通知。

## 4. 功能需求

### 4.1 登录认证

- 用户通过用户名和密码登录。
- 登录成功后返回访问 Token。
- 前端请求业务接口时携带 Token。
- 未登录访问业务页面时跳转登录页。
- 支持退出登录。
- 支持获取当前登录用户、角色和权限。

### 4.2 用户和权限

- 系统维护用户信息。
- 系统维护角色信息。
- 系统维护权限点。
- 用户可绑定多个角色。
- 角色可绑定多个权限。
- 第一版内置角色：管理员、测试人员、开发人员。

### 4.3 Bug 创建

- Bug 必须在需求下创建。
- 创建时后端根据需求自动关联项目。
- Bug 编号由系统自动生成。
- 创建人取当前登录用户。
- 支持填写：
  - 标题
  - 描述
  - 用例集，可选
  - 严重级别
  - 优先级
  - 指派人
- 创建成功后生成操作历史。
- 如果设置了指派人，给指派人发送站内通知。

### 4.4 Bug 查询

- 支持查询 Bug 列表。
- 支持按以下条件筛选：
  - 项目
  - 需求
  - 用例集
  - 状态
  - 严重级别
  - 指派人
  - 关键词
- 列表展示：
  - Bug 编号
  - 标题
  - 需求
  - 严重级别
  - 优先级
  - 状态
  - 创建人
  - 指派人
  - 图片数量
  - 更新时间

### 4.5 Bug 详情和编辑

- 支持查看 Bug 详情。
- 详情展示：
  - 基本信息
  - 图片附件
  - 评论流
  - 操作历史
- 支持编辑：
  - 标题
  - 描述
  - 状态
  - 严重级别
  - 优先级
  - 指派人
  - 关联用例集
- 关键字段变更需要写入操作历史。
- 状态变更、指派人变更需要触发站内通知。

### 4.6 图片上传

- Bug 支持上传多张图片。
- 支持格式：png、jpg、jpeg、gif、webp。
- 单张图片大小限制建议为 10MB。
- 上传后图片展示在 Bug 详情中。
- 图片支持预览和下载。
- 上传图片需要写入操作历史。

### 4.7 评论流

- 用户可在 Bug 下发表评论。
- 评论按创建时间正序展示。
- 评论展示评论人、评论时间和内容。
- 新增评论需要写入操作历史。
- 新增评论后通知 Bug 创建人和指派人，排除评论者本人。
- 第一版评论只支持纯文本。

### 4.8 操作历史

记录关键变更，不做完整字段审计。

需要记录：

- 创建 Bug。
- 修改标题。
- 修改描述。
- 修改状态。
- 修改严重级别。
- 修改优先级。
- 修改指派人。
- 修改关联用例集。
- 上传图片。
- 新增评论。

历史展示：

- 操作类型。
- 操作字段。
- 旧值。
- 新值。
- 操作人。
- 操作时间。

### 4.9 站内通知

需要触发通知的场景：

- Bug 指派给用户。
- Bug 状态变更。
- Bug 被评论。
- 可选：Bug 上传图片。

通知能力：

- 查询当前用户通知列表。
- 查询未读数量。
- 标记单条已读。
- 全部标记已读。
- 点击通知可定位到对应 Bug。

### 4.10 外部通知预留

第一版不接真实外部渠道，但需要预留：

- 外部通知服务接口。
- 通知外部发送状态字段。
- 后续可接邮件、企微、钉钉等渠道。

## 5. 状态和枚举

### Bug 状态

- `OPEN`：待处理
- `IN_PROGRESS`：处理中
- `RESOLVED`：已解决
- `CLOSED`：已关闭

### 严重级别

- `LOW`：低
- `MEDIUM`：中
- `HIGH`：高
- `CRITICAL`：严重

### 优先级

- `LOW`：低
- `MEDIUM`：中
- `HIGH`：高

### 通知状态

- `UNREAD`：未读
- `READ`：已读

## 6. 权限需求

第一版权限点：

- `BUG_VIEW`：查看 Bug。
- `BUG_CREATE`：创建 Bug。
- `BUG_EDIT`：编辑 Bug。
- `BUG_ASSIGN`：指派 Bug。
- `BUG_COMMENT`：评论 Bug。
- `BUG_UPLOAD_IMAGE`：上传 Bug 图片。
- `BUG_CLOSE`：关闭 Bug。
- `USER_MANAGE`：用户管理。

权限规则：

- 管理员拥有全部权限。
- 测试人员可创建、编辑自己创建的未关闭 Bug、上传图片、评论、指派。
- 开发人员可查看指派给自己的 Bug、修改处理状态、评论。
- 关闭 Bug 需要 `BUG_CLOSE` 权限。

## 7. 非目标

第一阶段不做：

- 邮件、企微、钉钉真实发送。
- 复杂审批流。
- 图片删除。
- 图片压缩和裁剪。
- 富文本评论。
- 评论附件。
- 批量操作。
- 和用例执行结果自动联动。
- 项目级复杂数据权限。
